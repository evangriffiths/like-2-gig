# Like2Gig

Find upcoming gigs near you for artists in your Spotify 'Liked Songs' playlist.

## Local Development

### Prerequisites

- Node.js 22+
- pnpm

### Setup

1. Create a Spotify app at <https://developer.spotify.com/dashboard>
   - Add `http://127.0.0.1:5173/auth/callback` as a redirect URI

2. Create a `.env` file in the project root:

   ```bash
   CLIENT_ID=your_spotify_client_id
   CLIENT_SECRET=your_spotify_client_secret
   ```

3. Install dependencies and start:

   ```bash
   pnpm install
   pnpm dev
   ```

The client runs at `http://127.0.0.1:5173` and the server at `http://127.0.0.1:3001`.

## Deployment

Deployed on a Hetzner VPS (CX22, Ubuntu) at <https://like2gig.evangriffiths.org>.

### Server config

All server configuration lives in `deploy/`:

| File | Purpose | Deployed to |
|------|---------|-------------|
| `deploy/setup.sh` | One-time server bootstrap | Run manually on fresh VPS |
| `deploy/like2gig.service` | systemd service (Node app) | `/etc/systemd/system/` (via deploy hook) |
| `deploy/Caddyfile` | Reverse proxy + SSL + basic auth | `/etc/caddy/Caddyfile` (manual) |
| `deploy/post-receive` | Git deploy hook | `/opt/like2gig.git/hooks/` |
| `deploy/crontab` | Daily sync at 4am UTC | `crontab -e` on server |

### Fresh server setup

1. **Provision a VPS** on Hetzner with primary IPv4, add your SSH key.

2. **Run the setup script:**

   ```bash
   ssh root@<your-ip>
   # Copy deploy/setup.sh to the server and run it:
   bash setup.sh
   ```

3. **Edit `/opt/like2gig/.env`** with your Spotify `CLIENT_ID` and `CLIENT_SECRET`.

4. **Set up DNS** — add an A record pointing your subdomain to the server IP.

5. **Configure Caddy** — copy `deploy/Caddyfile` to `/etc/caddy/Caddyfile`, set the password hash with `caddy hash-password --plaintext <password>`, then `systemctl restart caddy`.

6. **Add the redirect URI** in the Spotify Developer Dashboard: `https://like2gig.evangriffiths.org/auth/callback`

7. **First deploy** from your local machine: `git push deploy main`

8. **Allow native builds** (first time only):

   ```bash
   # On server, in /opt/like2gig:
   pnpm rebuild better-sqlite3
   ```

9. **Set up cron** — edit `deploy/crontab` with your `CRON_SECRET`, then on the server: `crontab deploy/crontab`

### Environment variables

`/opt/like2gig/.env` on the server:

```bash
CLIENT_ID=your_spotify_client_id
CLIENT_SECRET=your_spotify_client_secret
SESSION_SECRET=<auto-generated by setup.sh>
CRON_SECRET=<auto-generated by setup.sh>
BASE_URL=https://like2gig.evangriffiths.org
```

### Deploying changes

Pushing to `main` on GitHub automatically triggers a deploy via GitHub Actions. The workflow SSHs into the server and pushes to the bare git repo, which runs the post-receive hook.

**One-time setup for CI:**

```bash
# Add your SSH private key as a GitHub secret
gh secret set DEPLOY_SSH_KEY < ~/.ssh/id_ed25519
```

Then just push to GitHub as normal:

```bash
git push origin main
```

**Manual deploy** (bypasses GitHub):

```bash
# One-time: add the deploy remote
git remote add deploy root@<your-ip>:/opt/like2gig.git

# Deploy directly
git push deploy main
```

### Copying the local database to the server

To avoid re-syncing from scratch:

```bash
# Checkpoint WAL so all data is in the main .db file
sqlite3 server/data/like2gig.db "PRAGMA wal_checkpoint(TRUNCATE);"

# Stop app, copy, clean WAL files, restart
ssh root@<your-ip> 'systemctl stop like2gig'
scp server/data/like2gig.db root@<your-ip>:/opt/like2gig/server/data/like2gig.db
ssh root@<your-ip> 'rm -f /opt/like2gig/server/data/like2gig.db-wal /opt/like2gig/server/data/like2gig.db-shm && systemctl start like2gig'
```
